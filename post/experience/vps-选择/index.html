VPS 选择

2018-11-01
vps

      • 搬瓦工

      • vultr
      • 最后

写在前面

至今(2018-11-01)，我用 vps 已经超过 2 年多了，而且我又喜欢折腾，所以也算是积累了一些经验。起初我只是使用 vps 搭建 vpn，后来工作时间长了，开始喜欢写写博客，整理所学的知识，于是对 vps 的要求就更高了一点。本博客主要记录我选择 vps 的一个心路历程和遇到的各种坑，欢迎大家交流。

搬瓦工

我买过的第一个 vps 就是搬瓦工，到目前为止搬瓦工的性价比一定是最高的，而且仅仅使用 vpn 翻墙的话，买最便宜的(一年19.9$，不知道现在有没有了)一定绰绰有余了。至于主机，我一般都选洛杉矶，以前做过测试，从一天的不同时间段来看，洛杉矶的 vps 延迟可以保存基本不变，服务也不容易挂掉。尤其下午五点开始，各地区的 vps 都有可能出现短暂 ping 不同的现象。不过，如果只是玩 vpn 的话，那这点其实可以忽略的。

  • 总结：如果只是玩 vpn 那么选搬瓦工中最便宜的 vps 就可以了，付款的时候可以搜索搬瓦工的优惠码，想获得更多的vps信息可以看 老左推荐的 vps，不过不要全信。

LOCVPS

关于 LOCVPS， 我只买过一个美国洛杉矶的 vps，具体配置是： 2 cpu 2G 内存 … 其它忽略，大概花了50多人民币(定价是68，有一个终身8折优惠的活动)，使用起来还是很流畅的，无论在上传还是下载，测试速度基本都在 2MB/s 以上 (实际测试比这个值大很多)，以这个价格我都觉得占了很大便宜。当我把博客搬到 vps 上的时候才发现，博客页面展示浏览速度还可以，但是当下载资源(比如我链接的书籍资源)的时候，速度极慢，基本可以和github上下载大文件的速度相媲美了。仔细查看 vps 参数后发现，我的硬盘是 hdd 的，并不是ssd，或许这是一个原因吧。仔细比较了一下，我话50多买了一个 vps， 虽然有 2 个核，但是实际使用的不到一个核(博客和vpn并不太耗费cpu资源)，2G 内存也是用不了(想用啊，如果建个网站，内存大点肯定好，但是硬盘跟不上)，最后就不说硬盘了，确实得不偿失。

另外还有一个小插曲，locvps 默认值给系统分配了 10G 硬盘空间，加入你购买的是 100g 容量的vps，你得自己把剩余的 90g 挂在到系统目录下。

  • 总结：我觉得 vps 资源分配不是太好(起点太高)，对于建站 + vpn，这里的套餐还是显得太贵，不过如果租来服务器搭集群或者做附加存储的话，还是可以考虑，只是不知道服务器稳定性怎么样，不过看看官网的界面，还是觉得不太靠谱。

vultr

vultr 这个真是不好评论，但这是我现在一直使用的 vps (一直想试试 linode 但没有信用卡，也不计划找代购)，目前我用 5$/mon 的套餐，主要用来搭建 vpn 和 博客，总体来书还是挺稳定的，博客中附带的一些资源下载速度也还不错(毕竟我选的服务器在美国啊)，但是每天 5 点左右服务器卡的连 ssh 都没法连接，有一次我一怒之下请求退款了，商家半天之内就把钱给我退回来了（服务还是可以的）。因为尝试过很多其他家vps，感觉都不能让我满意。最后发现，其实 vultr相比其它 vps 已经很不错了，如果选择了好的地区(这个地区也不是固定的，而且每天的延时都不一样，但是总体还是稳定的)，服务也算是稳定的。

不过我的博客是用 hugo 生成的静态页面，如果再加一个数据库的话（比如我试过再加一个MongoDB），那么数据库可能会不定期的崩掉（计划以后再加一个网站，且需要使用数据的库话，我就升级 10$/mon 的套餐）。

  • 总体来说，vultr 上如果只是搭建 vpn + 静态博客，我认为是性价比最高的了，一个月 5$ ，大概也就35元人民币，如果使用PayPal付款会有汇率，所以尽量用支付宝付款，不过还是建议申请一个 PayPal 账号，因为PayPal偶尔会送你 10$ 的优惠券，这 10$ 可以用来购买 vps。最后 vultr 首充有活动，充多少送多少，最多送 100$ 。

最后

  1. 不要太相信别人对于 vps 的推荐，毕竟 vps 是可以赚美刀的，所以总结出来的 vps 的缺点并不算真正的缺点。
  2. 找 vps 还是不要贪小便宜，找大点的主机商还是有保证的，多比较每家 vps 的套餐和价格，那么你就明白，花一定的钱可以买到哪个级别的的 vps 配置 和 哪个级别的 vps 服务。
  3. 其实买 vps 多数是为了搭建 vpn，如果只是想写静态博客的话，用 github 提供的仓库足够了，而且不用担心服务会挂掉。所以，还在给还在墙内的同学留一个 go版本的linux shadowsocks 服务端在这里下载，go版本的比python版本的好用且效率高(好用在，python版本的可能会因为python解析器版本不兼容，需要修改几行代码，而go版本直接就是二进制可执行文件)。shadowsocket 配置是一样的 json 文件(不了解百度复制一份就好),下载了上述连接提供的 服务端文件，直接执行 <可执行文件:shadowsocks-server> -c <配置文件: xxx.json> 即可。windows 客户端在这里下载
  4. 如果喜欢写博客，hugo 之类的静态博客生成软件是极度推荐的，给服务器的压力小(占内存和cpu少)。
