Nginx一台主机配置两个网站

2018-10-12
NGINX

      • 1.大致过程
      • 2.具体配置如下（以ubuntu下nginx配置为例）
      • 3. 最后

1.大致过程

nginx 监听固定端口（http是80端口，https是443端口），当客户端请求到达nginx时，nginx根据server name将请求转发到另一个ip或者端口进行处理。当处理结束之后，再把处理结果转发回客户端。

2.具体配置如下（以ubuntu下nginx配置为例）

  1. 切换目录到 sits-enabled($表示命令行，不属于命令的一部分)

    $cd  /etc/nginx/sites-enabled/

  2. 配置端口转发 打开 default文件进行如下配置

    server { # web 服务1
    listen 80;
    server_name xxx.com www.xxx.com; # 这一行填写域名
    location / {
            proxy_pass http://127.0.0.1:8888; # 要转发到的端口或域名(我这是转发到本地8888端口)
    }
}
server { #web 服务2
    listen 80;
    server_name xxx2.com www.xxx2.com; # 填写另一个域名
    location / {
            proxy_pass http://127.0.0.1:6666; # 将 http://www.xxx2.com 转发到本地的 6666 端口 (具体参数可以自己配)
    }
}

  3. 在对应端口处理转发的请求

    1. 在 sits-enabled目录下创建文件并命名web1（名字可以随便取）

      server { # 对应处理上述 web服务1 的请求，使用到了php
listen 8888;
root /xxx/xxx/xxx; # 对应 web1 的网站根目录
index index.php index.html index.htm index.nginx-debian.html; # index文件名
server_name 127.0.0.1; # server name，对应 web1配置中的proxy_pass 中的ip（一致即可，可以随意定制）
location ~ ^.+\.php { # 因为使用了php，所以配置请求再次转发到9000端口，由php解析器处理
        fastcgi_pass   127.0.0.1:9000;
        fastcgi_index  index.php;
        fastcgi_split_path_info ^(.+\.php)(.*)$;
        fastcgi_param  PATH_INFO $fastcgi_path_info;
        fastcgi_param  PATH_TRANSLATED $document_root$fastcgi_path_info;
        include        fastcgi_params;
        include        fastcgi.conf;
}
}

    2. 在 sites-enabled目录下创建文件并命名web2（名字随便）

      server { # 对应处理上述 web服务器2 的请求，这是一个静态网站，没有使用php
listen 6666; # 对应 web2配置中的proxy_pass 中的端口（一致即可，可以随意配置，但要确保端口位置没被占用，且端口小于等于65536）
root /xxx/xxx/blog; # 对应 web2 的网站跟目录
index index.html index.htm; # index文件名
server_name 127.0.0.1; # server name，对应 web2 配置中的proxy_pass 中的ip（一致即可，可以随意定制）
}

  4. 最后重新加载nginx配置

    $ sudo nginx -s reload

    如果重新加载配置没有生效，只能重启nginx

    $ sudo nginx -s stop
$ sudo nginx

3. 最后

  1. 上述过程中，nginx配置了很多个 server， 然后发生了很多次转发过程，如果不喜欢这种方法，可以尝试所有配置写到 nginx 配置文件夹下 /etc/nginx/sites-enabled/的default文件中，只需要两个server即可（这种方法理论上应该可行，只是我暂时没有尝试）。
  2. 以我上边那种方式配置可以保证各个网站不混淆，每加一个网站就在default里添加一个server，再在sites-enabled下创建一个文件，对应该网站的配置。
