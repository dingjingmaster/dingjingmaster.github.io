增加/插入数据

  • 创建集合 插入文档的时候，文档所属集合不存在，那么插入操作会创建集合
  • 主键分配 每个插入的文档都需要指定唯一的id（”_id”）作为主键，如果没有指定，那么mongodb驱动会自动为”_id”字段生成一个 ObjectId。
  • 原子性 mongoDB中的所有写操作，在单一文档层级上是原子的

常用的文档插入

函数 说明
db.collection.insertOne() 向集合插入单个文档
db.collection.insertMany() 向集合中插入多个文档
db.collection.insert() 向集合插入一个或多个文档 
/* 向 users 集合插入一个文档 */
db.users.insertOne(
   {
      name: "sue",
      age: 19,
      status: "P"
   }
)


/* 向 users 集合插入多个文档 */
db.users.insertMany(
   [
     { name: "bob", age: 42, status: "A", },
     { name: "ahn", age: 22, status: "A", },
     { name: "xi", age: 34, status: "D", }
   ]
)


/* insert 向 users 集合插入一个文档 */
db.users.insert(
   {
      name: "sue",
      age: 19,
      status: "P"
   }
)


/* insert 向 users 集合插入读个文档 */
db.users.insert(
   [
     { name: "bob", age: 42, status: "A", },
     { name: "ahn", age: 22, status: "A", },
     { name: "xi", age: 34, status: "D", }
   ]
)

查询文档

db.collection.find( <查询条件>, <查询结果返回字段>)

  • 结果返回一个指向匹配文档的 cursor

一般查询

等于条件
  • 查询条件可以用 {<field1>:<value1>, ...} 的表达式形式指定包含 field 且该field 等于指定 value的所有文档。
/* test 集合中查找 status 字段等于 "A" 的所有文档 */
db.test.find( { status: "A" } )
查询操作符
  • 查询条件可以用 { <field1>: { <operator1>: <value1> }, ... } 的表达式形式指定包含 field 字段且该field满足 { <operator1>: <value1> }条件的所有文档。
/* test 集合中查找 status 字段值为 "P" 或 "D" 的所有文档 */
db.test.find( { status: { $in: [ "P", "D" ] } } )
指定AND条件
  • 查询条件可以在集合文档的多个字段上指定条件。隐含地，一个逻辑的 AND 连接词会连接复合查询的子句，使得查询选出集合中匹配所有条件的文档。
/* 在 test 集合中检索 status 等于 "A" 并且 age 小于 30 是所有文档 */
db.test.find( { status: "A", age: { $lt: 30 } } )
指定OR条件
  • 通过使用 $or 操作符，你可以指定一个使用逻辑 OR 连接词连接各子句的复合查询选择集合中匹配至少一个条件的文档。
/* 在 test 集合中查找 status 等于 "A" 或 age 小于 30 的所有文档 */
db.test.find(
   {
     $or: [ { status: "A" }, { age: { $lt: 30 } } ]
   }
)
AND和OR
/* 在 test 集合中查找 status 等于 "A" 且 (age 小于30 或 type 等于 1)的所有文档 */
db.test.find(
   {
     status: "A",
     $or: [ { age: { $lt: 30 } }, { type: 1 } ]
   }
)

嵌入文档查询

文档精确匹配
  • 使用 { <field>: <value> } 并且 “” 为要匹配文档的查询文档，来指定匹配整个内嵌文档的完全相等条件.(要使)相等条件匹配上内嵌文档需要指定 包括字段顺序的精确匹配。
/* 查询 test 集合中匹配所有 favorites 字段是中只包含article 和 food 字段 且 (artist="Picasso" food="pizza") 的文档 */
db.test.find( { favorites: { artist: "Picasso", food: "pizza" } } )
字段精确匹配
  • 使用 dot notation 匹配内嵌文档中的特定的字段。内嵌文档中特定字段的相等匹配将筛选出集合中内嵌文档包含该指定字段并等于指定的值的文档。内嵌文档可以包含其他的字段
/* 查找 test 集合中，包含 favorites 字段，且该字段下 内嵌文档包含 artist 字段, 且字段值为 "Picasso" 的所有文档 */
db.test.find( { "favorites.artist": "Picasso" } )

数组查询

数组精确匹配
  • 要指定数组相等匹配，使用查询文档 { : } 其中 是匹配的数组。数组的相等匹配要求数组字段与指定的匹配数组 完全 相符，包括数组元素的顺序。 ```js /* 查询 test 集合中，包含 badges 字段 且 值为 [“blue”, “black”] */ db.test.find( { badges: [ “blue”, “black” ] } )

// 返回结果示例 { “_id” : 1, “name” : “sue”, “age” : 19, “type” : 1, “status” : “P”, “favorites” : { “artist” : “Picasso”, “food” : “pizza” }, “finished” : [ 17, 3 ] “badges” : [ “blue”, “black” ], “points” : [ { “points” : 85, “bonus” : 20 }, { “points” : 85, “bonus” : 10 } ] }

###### 匹配一个数组元素

- 等于匹配可以指定匹配数组中的单一元素。如果数组中至少 一个 元素包含特定的值，就可以匹配这些声明。

```js
/* 查询 test 集合中包含 badges 字段，且该字段数组中包含 "black" 的所有文档 */
db.test.find( { badges: "black" } )

// 返回结果示例
{
   "_id" : 6,
   "name" : "abc",
   "age" : 43,
   "type" : 1,
   "status" : "A",
   "favorites" : { "food" : "pizza", "artist" : "Picasso" },
   "finished" : [ 18, 12 ],
   "badges" : [ "black", "blue" ],
   "points" : [ { "points" : 78, "bonus" : 8 }, { "points" : 57, "bonus" : 7 } ]
}
匹配数组中指定元素
  • 等于匹配可以指定匹配数组某一特定所有或位置的元素，使用 dot notation。
/* 查询 test 集合中包含 badges 字段，且该字段第 0 个元素值为 "black" 的所有文档 */
db.test.find( { "badges.0": "black" } )

// 返回结果示例
{
   "_id" : 6,
   "name" : "abc",
   "age" : 43,
   "type" : 1,
   "status" : "A",
   "favorites" : { "food" : "pizza", "artist" : "Picasso" },
   "finished" : [ 18, 12 ],
   "badges" : [ "black", "blue" ],
   "points" : [ { "points" : 78, "bonus" : 8 }, { "points" : 57, "bonus" : 7 } ]
}
指定数组元素多个查询条件
  • 使用 $elemMatch 操作符为数组元素指定复合条件，以查询数组中至少一个元素满足所有指定条件的文档。
/* 查询 test 集合中，包含 finished 字段,且该字段值是数组,该数组中有原始 大于15 且 小于20 的所有文档  */
db.test.find( { finished: { $elemMatch: { $gt: 15, $lt: 20 } } } )

// 返回结果
{
   "_id" : 6,
   "name" : "abc",
   "age" : 43,
   "type" : 1,
   "status" : "A",
   "favorites" : { "food" : "pizza", "artist" : "Picasso" },
   "finished" : [ 18, 12 ],
   "badges" : [ "black", "blue" ],
   "points" : [ { "points" : 78, "bonus" : 8 }, { "points" : 57, "bonus" : 7 } ]
}
元素组合满足查询条件
/* 查询 test 集合中，包含 finished 字段,且该字段值是数组,该数组中有元素 大于15 且 也有元素 小于20 的所有文档 */
db.test.find( { finished: { $gt: 15, $lt: 20 } } )

// 返回结果
{
   "_id" : 2,
   "name" : "bob",
   "age" : 42,
   "type" : 1,
   "status" : "A",
   "favorites" : { "artist" : "Miro", "food" : "meringue" },
   "finished" : [ 11, 20 ],
   "badges" : [ "green" ],
   "points" : [ { "points" : 85, "bonus" : 20 }, { "points" : 64, "bonus" : 12 } ]
}
嵌入文档数组
  • 使用数组索引匹配嵌入文档中的字段
/* 查询 test 集合中，包含points的字段，且points是数组，查询该字段对于值第0个位置对象的 point value 小于55 的文档 */
db.test.find( { 'points.0.point': { $lte: 55 } } )

// 返回结果

  • 不指定数组素银匹配字段

如果你不知道文档在数组中的索引位置，用点号 (.) 将包含数组的字段的名字和内嵌文档的字段的名字连起来。

/* 查询 test 集合中，所有 points 数组中至少有一个嵌入文档包含值小于或等于 55 的字段 points 的文档 */
db.test.find( { 'points.points': { $lte: 55 } } )

多个查询条件
  • 单个元素满足查询条件

使用 $elemMatch 操作符为数组元素指定复合条件，以查询数组中至少一个元素满足所有指定条件的文档。

/* 查询 test 集合中， points 数组有至少一个包含 points 小于等于 70 并且字段 bonus 等于 20 的内嵌文档的文档 */
db.test.find( { points: { $elemMatch: { points: { $lte: 70 }, bonus: 20 } } } )

// 返回结果示例
{
   "_id" : 3,
   "name" : "ahn",
   "age" : 22,
   "type" : 2,
   "status" : "A",

   "finished" : [ 6 ],
   "badges" : [ "blue", "red" ],
   "points" : [ { "points" : 81, "bonus" : 8 }, { "points" : 55, "bonus" : 20 } ]
}
  • 元素组合满足查询条件
/* 查询 test 集合中，points 数组包含了以某种组合满足查询条件的元素的文档 */
db.test.find( { "points.points": { $lte: 70 }, "points.bonus": 20 } )

// 结果

   "_id" : 2,
   "name" : "bob",
   "age" : 42,

字段过滤
  • 获取文档所有字段的话，可以湖绿该参数，若保留结果中的字段则将该字段置为 true 或 1，反之置为 false 或 0
/* 查找 test 集合中 status 为 "A" 的所有文档，且返回这些文档的 name 字段 和 _id 字段，不反回 status 字段 */
db.test.find( { status: "A" }, { name: 1, status: false } )
  • 返回嵌入文档中的指定字段
/* 查找 test 集合中 status 为 "A" 的所有文档，只返回 name、_id、status、favorite 下的 foot 字段 */
db.test.find(
   { status: "A" },
   { name: 1, status: 1, "favorites.food": 1 }
)
  • 排除嵌入文档中的特定字段
/* 查找 test 集合中 status 为 "A" 的所有文档，除了 favorite 下的 food 字段，其它字段都返回 */
db.test.find(
   { status: "A" },
   { "favorites.food": 0 }
)
  • 映射数组中的嵌入文档¶
/* 映射返回 name 字段、 status 字段，以及 points 数组的文档中包含 bonus 字段的文档。 默认返回 _id */
db.test.find( { status: "A" }, { name: 1, status: 1, "points.bonus": 1 } )
  • 映射返回数组中特定的数组元素
/* 使用 $slice 映射操作符来返回 scores 数组中最后的元素 */
db.test.find( { status: "A" }, { name: 1, status: 1, points: { $slice: -1 } } )

查找空值或不存在字段

查找不存在字段或字段值为空
/* 返回 name 字段是空 或 name 字段不存在的文档 */
db.test.find( { name: null } )
类型筛查
/* { name : { $type: 10 } } 查询 仅仅 匹配那些包含值是 null 的 name 字段的文档,亦即 条目 字段的值是BSON类型中的 Null (即 10 ) */
db.test.find( { name : { $type: 10 } } )
存在性筛查
/* 查找不包含 name 字段的 文档 */
db.test.find( { name : { $exists: false } } )

查询结果迭代

手动迭代 cursor

  • 在 mongo shell 中，没有把查询结果赋值给变量，则会自动迭代前20条结果，否则需要手动迭代游标
var myCursor = db.users.find( { type: 2 } );
while (myCursor.hasNext()) {
   print(tojson(myCursor.next()));
}

或

var myCursor =  db.users.find( { type: 2 } );
myCursor.forEach(printjson);

迭代器索引

  • 在 mongo shell，可以使用 toArray() 方法来迭代游标，并且以数组的形式来返回文档
var myCursor = db.inventory.find( { type: 2 } );
var documentArray = myCursor.toArray();
var myDocument = documentArray[3];

或直接在游标上使用索引来访问文档

var myCursor = db.users.find( { type: 2 } );
var myDocument = myCursor[1];

游标的行为

关闭游标
  • 默认情况下，服务在游标使用 10 分钟后 或者 客户端用尽游标后关闭游标，可以使用(cursor.noCursorTimeout()防止过时后关闭游标)
var myCursor = db.users.find().noCursorTimeout();
  • 使用 cursor.close() 手动关闭游标
隔离游标
  • 当游标返回文档时，其他操作可能与查询交织在一起。对于MMAPv1存储引擎，如果文档发生了更改，那么在文档上进行插入操作可能会导致光标返回文档不止一次。
游标批次
  • MongoDB 服务分批返回游标，批处理中的数据量不会超过BSON文档的最大大小。
  • 默认情况下，find()和aggregate()操作的批处理大小为101个文档。针对生成的游标发出的后续getMore操作没有默认的批处理大小，因此它们仅受16兆字节消息大小的限制。
  • 对于包含没有索引的排序操作的查询，服务器必须在返回任何结果之前加载内存中的所有文档来执行排序。
  • 当您遍历游标并到达返回批处理的末尾时，如果有更多的结果，那么cursor.next()将执行getMore操作来检索下一个批处理。
var myCursor = db.inventory.find(); // 返回游标
var myFirstDocument = myCursor.hasNext() ? myCursor.next() : null; // 获得游标第一个元素
myCursor.objsLeftInBatch(); // 迭代游标中，批处理中还剩多少文档
游标信息
  • 使用 db.serverStatus() 获得游标信息的度量文档， 文档包含以下信息：
    1. 自上次服务器重新启动以来超时游标的数量
    2. 通过 DBQuery.Option.noTimeout 设置了在一段时间不活动之后防止超时断开的游标数量
    3. 固定打开游标的书目
    4. 游标的总数
/* 获取游标信息 */
db.serverStatus().metrics.cursor

// 返回结果
{
   "timedOut" : <number>
   "open" : {
      "noTimeout" : <number>,
      "pinned" : <number>,
      "total" : <number>
   }
}

更新文档

操作 说明
db.collection.updateOne() 即使可能有多个文档通过过滤条件匹配到，但是也最多也只更新一个文档。
db.collection.updateMany() 更新所有通过过滤条件匹配到的文档.
db.collection.replaceOne() 即使可能有多个文档通过过滤条件匹配到，但是也最多也只替换一个文档。
db.collection.update() 即使可能有多个文档通过过滤条件匹配到，但是也最多也只更新或者替换一个文档。
  • 默认情况下, db.collection.update() 只更新 一个 文档。要更新多个文档，请使用 multi 选项。
  • 更新文档意味着，除了 “_id” 其它字段都会替换
/* 更新操作 */
db.collection.update(
   <query>, // 查找指定文档的条件
   <update>, // 要更新的值
   { // 更新相关参数，后续...
     upsert: <boolean>,
     multi: <boolean>,
     writeConcern: <document>
   }
)
  • 查询过滤文档 能够用 : 表达式指定相等条件并以此选出所有包含有指定 的 的文档
{ <field1>: <value1>, ... }
  • 查询过滤文档 能使用以下 ref:查询操作符 来指定查询条件
{ <field1>: { <operator1>: <value1> }, ... }

更新行为

原子性
  • MongoDB中所有的写操作在单一文档层级上是原子的
_id 字段
  • 一旦设定，不能更新 _id 字段的值，也不能用有不同 _id 字段值的替换文档来替换已经存在的文档。
文档大小
  • 当执行更新操作增加的文档大小超过了为该文档分配的空间时。更新操作会在磁盘上重定位该文档。
字段顺序
  • MongoDB按照文档写入的顺序整理文档字段，除了如下的情况：
    1. _id 字段始终是文档中的第一个字段。
    2. 包括字段名称的 renaming 操作可能会导致文档中的字段重新排序。
upsert 选项
  • 更新文档时，若操作包含 upsert : true 并且 没有文档匹配指定的过滤器，那么此操作会创建一个新文档并插入它。如果有匹配的文档，那么此操作修改或替换匹配的单个或多个文档。

删除文档

操作 说明
db.collection.remove() 删除一个或多个匹配过滤条件的文档
db.collection.deleteOne() 即使多个文档可能与指定筛选器匹配，也最多删除其中一个匹配的文档
db.collection.deleteMany() 删除所有匹配指定过滤条件的文档
  • 可以指定条件或过滤器来找到要删除的文档.这些 filters 使用与 查找 操作语法相同

删除的行为表现

索引
  • 删除操作不会删除索引
原子性
